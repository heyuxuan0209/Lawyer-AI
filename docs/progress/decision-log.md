# 决策日志

## 2026-05-19：第一阶段先做客户咨询，而不是文书、资讯、营销或完整办案

决策：

优先切入客户咨询场景，产品定位为“客户咨询助手 / 谈案助手”。

原因：

- 客户咨询离律师成交最近。
- 输入天然存在，包括录音、微信记录、会议纪要、客户材料、律师备注。
- 风险更可控，AI 只做律师复核草稿，不直接给客户最终法律意见。
- 后续可自然延展到法律意见书、律师函、起诉状、证据目录和案件工作台。

## 2026-05-19：MVP 先做劳动争议

决策：

第一个 MVP 选择劳动争议付费咨询。

原因：

- 劳动争议高频。
- 事实结构相对清楚。
- 材料类型明确，如劳动合同、工资流水、社保记录、解除通知、考勤记录、聊天记录。
- 容易形成标准问诊表、材料清单和法律意见书模板。

## 2026-05-19：第一版先做咨询后整理，不做实时语音

决策：

第一版支持粘贴咨询记录、录音转写文本、微信聊天记录和律师备注，暂不做实时录音转写。

原因：

- 核心验证点是输出质量，不是实时能力。
- 咨询后整理更容易让律师接受，风险更低。
- 可以更快做出 MVP，拿真实脱敏材料验证。

## 2026-05-19：采用三栏式工作台

决策：

参考 `Mall Operation Agent` 的 `/workspace` 信息组织方式，法律咨询助手也采用三栏式工作台。

映射关系：

```text
经营事项线程 -> 咨询线程
会商消息流 -> 咨询处理流
证据账本/决策栏 -> 案情结构化/交付件/律师确认
```

原因：

- 一级对象是“咨询线程”，不是后台菜单。
- 中间区域适合呈现从原始咨询到 AI 整理的过程。
- 右侧适合放律师确认和可交付产物。

## 2026-05-19：方向修正，从咨询后整理转为实时谈案 Copilot

决策：

主产品方向从“咨询后整理 / 交付件生成”调整为“实时谈案 Copilot”。

原因：

- 律师实际需求不是先看会后整理产物，而是在咨询过程中完成语音转记。
- 律师希望系统能边聊边形成方案，而不是咨询结束后才生成结果。
- 律师还希望系统能在过程中推送相关法律条款，辅助其判断和表达。

新主线：

```text
语音转记
-> 实时案情要点提取
-> 实时追问建议
-> 实时法律条款推送
-> 过程中形成方案草稿
-> 咨询结束后沉淀纪要和材料清单
```

对旧版本的处理：

- 咨询纪要、案情结构化、材料清单、初步法律分析继续保留。
- 这些能力从“第一屏主产品”降级为“咨询结束后的沉淀产物”。
- 法律意见书草稿后置，不作为实时 MVP 的核心卖点。
