# 实时谈案 Copilot PRD v0.6

更新时间：2026-05-19

## 1. 产品目标

把劳动争议咨询场景从“咨询后整理”推进为“咨询中实时辅助”。

第一版目标不是做完整法律系统，而是验证：

```text
律师在真实咨询过程中，是否需要一个能实时转记、抓重点、提醒追问、推送法条、形成方案的 Copilot。
```

## 2. 用户与场景

用户：

- 面向劳动争议咨询的律师
- 需要付费咨询留痕和实时辅助的律师

使用场景：

- 电话咨询
- 线上会议咨询
- 线下面谈咨询

典型流程：

```text
律师点击开始咨询
-> 客户开始陈述
-> 系统实时转写
-> 系统识别案情要点
-> 系统推送追问建议和法律条款
-> 律师边问边修正方案
-> 咨询结束后系统自动沉淀纪要和材料清单
```

## 3. MVP 功能

### 3.1 实时转记区

第一版模拟转写，不接真实语音。

功能：

- 点击“开始咨询”
- 客户陈述逐段出现
- 支持客户/律师角色区分
- 关键事实自动高亮
- 重要信息可标记为“已确认”

### 3.2 案情要点面板

实时更新：

- 入职时间
- 岗位
- 工资标准
- 劳动合同情况
- 解除时间
- 解除理由
- 解除方式
- 现有证据
- 缺失证据

### 3.3 追问建议面板

根据当前转写推送：

- 是否签劳动合同？
- 是否收到书面解除通知？
- 是否签过绩效考核结果？
- 是否有绩效改进计划？
- 是否签署离职结算文件？
- 是否有工资流水、考勤、社保记录？

每条追问支持反馈：

- 有用
- 暂不需要
- 错误

### 3.4 法律条款推送面板

第一版先用模拟法条卡，不接真实检索。

法条卡字段：

- 条款主题
- 相关原因
- 适用前提
- 风险提示
- 核验状态

注意：

- 不输出未经核验的具体条号作为确定依据。
- 可展示“需进一步核验”的法条提示。
- 第一版重点是验证法条推送时机和形式。

### 3.5 方案草稿面板

随着咨询推进持续更新：

- 初步判断
- 处理路径
- 证据补强方向
- 是否建议进入付费深入咨询/委托
- 风险话术

### 3.6 结束咨询沉淀

点击“结束咨询”后生成：

- 咨询纪要
- 案情结构化
- 待补材料清单
- 初步方案摘要

法律意见书草稿暂时后置，不作为 v0.7 第一视觉重点。

## 4. 页面结构

### 左栏：咨询线程

```text
正在咨询
待补材料
待出方案
已结束
```

### 中栏：实时转写

```text
客户发言
律师发言
AI 识别标签
关键事实高亮
咨询阶段进度
```

### 右栏：实时辅助

```text
案情要点
建议追问
相关法律条款
初步方案
风险提示
结束后产物
```

## 5. 第一版演示脚本

劳动争议违法解除咨询。

模拟客户分段陈述：

1. 入职时间、岗位、工资。
2. 公司以绩效不达标要求离开。
3. 没有书面解除通知和绩效改进计划。
4. 有劳动合同、工资流水、微信记录，考勤不完整。
5. 想要赔偿金、年假工资和加班费。

每段触发右侧更新：

- 案情要点
- 追问建议
- 法条提示
- 方案草稿

## 6. 不做范围

v0.7 不做：

- 真实录音
- 真实实时 ASR
- 真实法条检索
- 类案检索
- 客户门户
- 账号权限
- Word 导出
- 多案由切换

## 7. 验证指标

拿给律师看时，重点问：

1. 实时转写区是否符合咨询过程？
2. 右侧追问建议是否有用，还是干扰？
3. 法条推送是否应该出现，出现时机是否合适？
4. 方案草稿是否能帮助律师组织表达？
5. 咨询结束后的纪要和材料清单是否仍有价值？

成功信号：

```text
律师认为实时追问建议和案情要点比会后意见书更有价值；
律师愿意用自己的真实咨询音频/转写测试；
律师能指出哪些法条推送有用，哪些会干扰。
```
