# v0.8 律师演示验证清单

更新时间：2026-05-19

## 目标

本清单用于拿实时谈案 Copilot 给律师朋友演示时收集反馈。

不要泛泛问“这个产品怎么样”，而是让律师围绕真实咨询流程判断：

```text
实时转写是否像咨询现场？
右侧辅助是否有用？
哪些提示会干扰？
哪些产物值得保留？
```

## 演示前说明

先向律师明确：

- 这是劳动争议咨询场景的模拟原型。
- 当前不接真实麦克风。
- 法条卡是模拟卡片，只验证推送时机和展示形式。
- 法律意见书是模拟格式样例，未经律师审核不可发送客户。

## 演示步骤

1. 打开 `prototype/index.html`。
2. 点击“开始咨询”。
3. 让对话自动流式播放。
4. 观察右侧案情要点、追问建议、可能相关规则和方案草稿是否同步更新。
5. 点击“结束咨询”。
6. 依次查看咨询纪要、材料清单、方案摘要、模拟法律意见书格式样例。

## 必问问题

### 1. 实时转写流

- 这段客户表达像不像你真实咨询里遇到的客户？
- 客户/律师分角色对话是否符合你的使用习惯？
- 如果是真实咨询，你会希望 AI 自动标记哪些原话？

### 2. 案情要点

- 右侧案情要点是否抓住了劳动争议判断所需字段？
- 有没有必须前置但现在没显示的字段？
- 字段更新速度是刚好、太快，还是太慢？

### 3. 追问建议

- 哪些追问建议是你确实会问的？
- 哪些追问建议没必要，甚至会打断你？
- 追问建议应该显示几条最合适？

### 4. 模拟法条卡

- 法条卡应该在什么时候出现？
- 当前“可能相关规则 / 为什么推送 / 适用前提 / 需核验”的格式是否合适？
- 是否应该显示具体法条号，还是先显示规则主题？

### 5. 方案草稿

- 方案草稿是否能帮助你边聊边组织判断？
- 它应该更偏律师内部思路，还是可以整理成客户能听懂的话？
- 哪些表达有法律风险？

### 6. 会后产物

- 咨询纪要有没有价值？
- 材料清单能不能直接发客户？
- 方案摘要是否比法律意见书更适合第一版？
- 模拟法律意见书格式是否接近你的交付格式？
- 如果只能保留一个会后产物，你选哪个？
