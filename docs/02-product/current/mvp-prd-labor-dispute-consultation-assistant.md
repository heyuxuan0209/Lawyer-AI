# 劳动争议咨询助手 MVP PRD

更新时间：2026-05-19

## 1. MVP 定位

产品名称暂定：

```text
劳动争议付费咨询助手
```

一句话定义：

> 把一次劳动争议付费咨询的录音转写、微信记录或律师备注，自动整理成咨询纪要、案情结构化、待补材料清单、初步法律分析和法律意见书草稿，由律师确认后使用。

第一版不是：

- 面向客户的法律问答机器人
- 自动给出最终法律意见的系统
- 完整办案系统
- 实时录音转写产品
- 法条和类案检索平台

第一版是：

- 咨询后整理工具
- 律师内部工作底稿生成工具
- 付费咨询留痕工具
- 法律意见书草稿生成工具

## 2. 目标用户

主要用户：

- 处理劳动争议咨询的律师
- 有付费咨询、咨询留痕、意见书交付需求的律师

典型使用场景：

```text
客户在线上/线下咨询中讲述劳动争议情况
-> 律师口头澄清并给出初步判断
-> 咨询结束后，律师将转写文本/微信记录/备注放入系统
-> AI 生成咨询纪要和法律意见书草稿
-> 律师修改确认后留档或交付客户
```

## 3. 核心问题

律师在劳动争议咨询中面临的问题：

- 客户陈述时间长，信息分散。
- 律师需要快速抓住核心事实。
- 咨询后整理会议纪要耗时。
- 付费咨询需要留痕。
- 法律意见书格式固定，但从口头咨询整理成书面内容很耗时。
- 客户材料经常不完整，需要整理待补材料清单。

MVP 要验证：

```text
AI 是否能把混乱的客户咨询，稳定整理成律师愿意修改和使用的结构化工作底稿。
```

## 4. 核心流程

```text
新建咨询
-> 选择咨询类型：劳动争议
-> 粘贴咨询记录 / 录音转写 / 微信记录 / 律师备注
-> AI 生成咨询纪要
-> AI 生成案情结构化
-> AI 生成待补材料清单
-> AI 生成初步法律分析
-> AI 生成法律意见书草稿
-> 律师修改确认
-> 导出 / 留档
```

## 5. 页面结构

采用三栏式工作台。

### 5.1 顶部栏

包含：

- 产品名：律师谈案助手
- 当前场景：劳动争议咨询
- 搜索框：搜索客户、咨询、争议类型
- 主按钮：新建咨询
- 状态提示：AI 草稿 / 待律师确认 / 已确认

### 5.2 左栏：咨询线程列表

左栏是咨询线索和咨询线程，不是功能菜单。

分组：

- 待整理
- 待补材料
- 待律师确认
- 已生成意见书
- 已结束

咨询卡片字段：

- 客户姓名
- 争议类型
- 状态
- 咨询方式
- 咨询时长
- 最近更新时间
- 是否付费
- 是否有意见书草稿

示例：

```text
张某｜违法解除
付费咨询 · 42分钟
待律师确认
缺少：解除通知、工资流水
```

### 5.3 中栏：咨询处理流

中栏承载从原始咨询到 AI 整理的过程。

内容块：

1. 原始咨询记录
2. AI 咨询纪要
3. AI 关键事实提取
4. AI 缺失问题提醒
5. 律师补充备注
6. AI 初步法律分析
7. AI 法律意见书草稿

中栏应像“线程”，让律师看到处理过程，而不是只看到一个表单。

### 5.4 右栏：案情结构化与交付件

右栏是律师判断和交付的核心区域。

模块：

- 当前案情状态
- 案情结构化
- 争议焦点
- 待补材料清单
- 风险提示
- 交付件列表
- 律师确认区

状态：

```text
AI 草稿
待律师确认
已确认
可导出
```

## 6. 第一版功能范围

### 6.1 必做

- 新建劳动争议咨询
- 粘贴咨询文本
- 内置一条示例咨询记录
- 生成咨询纪要
- 生成案情结构化表
- 生成待补材料清单
- 生成初步法律分析
- 生成法律意见书草稿
- 标记“待律师确认”

### 6.2 暂不做

- 实时录音
- 实时转写
- 客户门户
- 在线支付
- 账号权限系统
- 法条数据库深度接入
- 类案检索
- 正式案件管理
- 多案由支持
- Word 格式精排

## 7. 劳动争议字段

### 7.1 基础信息

- 客户姓名
- 联系方式
- 咨询时间
- 咨询方式
- 是否付费咨询
- 咨询时长

### 7.2 劳动关系信息

- 用人单位名称
- 入职时间
- 离职 / 解除时间
- 岗位
- 工资标准
- 工资发放方式
- 劳动合同签署情况
- 社保缴纳情况

### 7.3 争议类型

- 违法解除
- 拖欠工资
- 未签劳动合同
- 未缴社保
- 加班费
- 经济补偿 / 赔偿金
- 工伤
- 其他

### 7.4 证据材料

- 劳动合同
- 工资流水
- 社保记录
- 解除通知
- 考勤记录
- 工作沟通记录
- 录音录像
- 工牌 / 工作证
- 入职通知
- 离职证明

## 8. 输出模板

### 8.1 咨询纪要

```text
一、咨询基本信息
二、客户主要诉求
三、事实经过
四、现有材料
五、律师初步回应
六、待补充事项
七、后续安排
```

### 8.2 案情结构化表

```text
客户诉求：
劳动关系：
关键时间线：
争议焦点：
现有证据：
缺失证据：
初步风险：
下一步动作：
```

### 8.3 待补材料清单

```text
必须补充：
- ...

建议补充：
- ...

无法提供时需说明：
- ...
```

### 8.4 初步法律分析

```text
一、可能法律关系
二、可能主张
三、初步支持因素
四、初步不利因素
五、需进一步核验的问题
六、不得直接承诺的事项
```

### 8.5 法律意见书草稿

```text
一、基本事实
二、咨询问题
三、初步法律分析
四、处理建议
五、材料补充清单
六、风险提示
七、律师确认
```

## 9. 示例咨询线程

MVP 内置一条演示数据：

```text
客户张某，2023年3月入职某科技公司，岗位为运营，月工资12000元。
2026年4月公司以“绩效不达标”为由口头通知解除劳动关系。
客户称没有收到书面解除通知，也没有签过正式绩效改进计划。
公司只支付到4月底工资，未支付经济补偿。
客户有劳动合同、工资流水、部分微信沟通记录，但没有完整考勤记录。
客户希望了解能否主张违法解除赔偿金、未休年假工资和加班费。
```

系统应生成：

- 咨询纪要
- 案情结构化表
- 待补材料清单
- 初步法律分析
- 法律意见书草稿

## 10. 验证指标

拿给律师朋友测试时，不问“你觉得产品怎么样”，而是让对方评价具体产物：

- 咨询纪要是否覆盖主要事实
- 案情结构化是否抓住关键点
- 待补材料清单是否准确
- 初步分析是否有危险表达
- 法律意见书草稿是否可修改后使用
- 是否节省咨询后整理时间
- 是否愿意继续用第二条真实咨询记录测试

MVP 成功信号：

```text
律师愿意提供 2-3 条脱敏咨询记录继续测试；
生成内容覆盖 70% 以上整理工作；
律师愿意在真实付费咨询后使用；
律师认为输出能作为内部底稿或初稿。
```

## 11. 下一步开发建议

第一步做静态三栏 Demo：

- 左侧显示 3 条咨询线程
- 中间显示劳动争议咨询处理流
- 右侧显示案情结构化和交付件

第二步加入输入能力：

- 支持粘贴咨询文本
- 点击按钮生成模拟结果

第三步接入真实 AI：

- 用固定 Prompt 生成四类输出
- 将输出结果保存为 Markdown

第四步做律师反馈闭环：

- 每个输出块支持“可用 / 需修改 / 有风险”
- 收集律师反馈
- 迭代 Prompt 和模板
