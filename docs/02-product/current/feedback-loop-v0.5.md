# v0.5 Feedback Loop：律师反馈闭环

更新时间：2026-05-19

## 目标

在演示原型中加入结构化反馈能力，让律师朋友不只是口头评价“好不好”，而是能逐个产物标记：

```text
可用 / 需修改 / 有风险
```

## 为什么需要反馈闭环

法律咨询助手的关键不是一次生成看起来完整，而是持续发现：

- 哪些产物最有价值
- 哪些内容律师愿意直接保留
- 哪些内容需要律师重写
- 哪些表达存在法律风险
- 风险来自事实遗漏、证据不足、法律判断过度，还是对外话术不合适

## v0.5 原型变化

在每个 AI 产物卡片底部增加反馈条：

- 可用
- 需修改
- 有风险

右侧增加反馈汇总：

- 可用数量
- 需修改数量
- 有风险数量
- 最近反馈记录

## 给律师朋友的使用方式

让律师按顺序看 5 个产物：

1. 咨询纪要
2. 案情结构化
3. 待补材料清单
4. 初步法律分析
5. 法律意见书草稿

每看完一个，立即标记：

- 如果基本可保留，点“可用”
- 如果方向对但需要改，点“需修改”
- 如果存在不应出现的判断或表达，点“有风险”

## 后续应补充

当前 v0.5 只做前端演示，尚未持久化反馈。

后续版本建议：

- 增加反馈备注输入框。
- 将反馈保存到 `data/outputs` 或本地 JSON。
- 按产物类型统计风险来源。
- 用反馈反推 Prompt 迭代。
