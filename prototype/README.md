# Prototype

当前只保留一个演示入口：

```text
prototype/index.html
```

注意：不要只单独发送 `index.html`。页面还依赖同目录下的：

```text
prototype/styles.css
prototype/app.js
```

如果要发给律师本地打开，建议把整个 `prototype` 文件夹压缩后发送，对方解压后打开 `index.html`。

该页面就是实时谈案 Copilot 原型：

```text
劳动争议咨询
-> 客户 / 律师分角色自动流式转写
-> 右侧同步更新案情要点、追问建议、模拟法条卡、方案草稿
-> 打开“反馈模式”后，律师可当场标记追问、法条卡、方案草稿是否有用
-> 结束咨询后生成咨询纪要、材料清单、方案摘要、模拟法律意见书格式样例
```

当前版本重点给律师验证：

```text
1. 实时转写流是否像真实咨询现场
2. 追问建议是否帮忙，还是干扰谈案
3. 模拟法条卡的出现时机和颗粒度是否合适
4. 方案草稿是否能帮助律师边聊边组织处理路径
5. 会后产物中哪些值得进入下一版
```

归档原型放在：

```text
prototype/archive/
```

演示时不要打开归档目录，直接打开 `prototype/index.html`。
